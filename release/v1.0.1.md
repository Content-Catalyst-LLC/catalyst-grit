# Catalyst Grit v1.0.1

**Release:** Repository Integrity and Product Consolidation  
**Version:** 1.0.1

## Outcome

Catalyst Grit now has one production product model: the structured recovery record. The obsolete event-count and trait-metrics Flask prototype is isolated under `legacy/` and excluded from the installed package, API contract, production tests, and WordPress artifact.

## Included

- Canonical recovery-record package API and command-line interface
- Unified `1.0.1` identity across package, plugin, schema, manifest, OpenAPI, browser engine, documentation, and examples
- Versioned schema and engine provenance in generated records
- Python/browser golden-fixture parity tests
- JSON Schema, package build/install, PHP, JavaScript, output, and repository-integrity validation
- Removal of tracked SQLite and generated package metadata
- Deterministic source and WordPress packaging with checksums and release manifest
- macOS safety-backup, install, validate, commit, and push workflow

## Compatibility

The v1.0.0 command `python3 python/catalyst_grit_core.py ...` remains available as a wrapper around the canonical installed package. The public WordPress shortcode remains `[catalyst_grit_demo]`.

## Boundaries

Catalyst Grit does not diagnose people, score character, rank employees, provide mental-health advice, or guarantee outcomes.
