# Catalyst Grit v1.2.0 — Persistent Records, Projects, and Review Checkpoints

This release turns the canonical recovery-record calculator into a durable private workspace.

## Added

- SQLite repository abstraction with two ordered reversible migrations.
- Private recovery projects and records.
- Append-only record revisions preserving canonical output, exact input snapshots, versions, hashes, actors, and reasons.
- Revision-scoped action snapshots.
- Review checkpoints, review events, status history, and audit events.
- Reopen, revise, compare, duplicate, archive, soft-delete, retention, and guarded purge workflows.
- v1.0/v1.1/v1.2 record import and workspace bundle export/import.
- Workspace health, integrity, migration, and entity-count diagnostics.
- CLI commands covering database, project, record, checkpoint, review, retention, and bundle operations.
- Authenticated WordPress private workspace with nonce and per-user storage.
- Separate public and private WordPress assets; the public demo remains non-persistent.

## Boundary

Persistence does not change Catalyst Grit into a diagnostic, character, employee-ranking, or automated performance system. Records remain private by default and consequential interpretation requires human review.

## Installer repair — REPAIRED_V2

The macOS installer now runs the dependency-free smoke test with isolated Python (`-I -S`) and clears `PYTHONPATH`/`PYTHONHOME`. The smoke test also verifies that `catalyst_grit` was imported from the newly installed repository source and reports the actual module path on failure. This prevents older editable or globally installed package copies from causing a false version identity mismatch.

## Installer repair — REPAIRED_V3

The installer now uses `rsync -a --checksum --delete` rather than archive-mode timestamp/size quick checks. Deterministic release files intentionally share fixed timestamps, and version strings such as `1.1.0` and `1.2.0` have equal byte lengths; the former command could therefore skip changed files. Post-copy `VERSION` and Python version-module comparisons now fail before testing if installation did not reproduce the release source. Isolated Python smoke validation from REPAIRED_V2 remains enabled.
