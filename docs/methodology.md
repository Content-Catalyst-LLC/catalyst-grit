# Catalyst Grit Methodology

Catalyst Grit treats resilience as a system pattern rather than a personality trait. The module asks what happened, which pressures were present, how severe the impact was, what supports exist, what clarity remains, and which recovery actions are feasible.

## Core idea

Progress is rarely linear. A useful resilience workflow should help people and teams recover faster, learn honestly, and adjust the system that produced the friction.

## Review path

```text
setback → context → impact → pressure → support → clarity → recovery action → learning → next step
```

## Design principles

1. **No character scoring.** Grit is not a moral label.
2. **Context matters.** Pressure, constraints, and support change recovery capacity.
3. **Recovery is observable.** The useful question is what helps the system regain motion.
4. **Learning loops matter.** Repeated setbacks should inform design, planning, support, and workload.
5. **Human judgment remains responsible.** The module structures reflection; it does not decide for the user.

## Boundaries

Catalyst Grit is educational and analytical. It is not a medical, mental-health, HR, coaching, legal, or compliance tool.
